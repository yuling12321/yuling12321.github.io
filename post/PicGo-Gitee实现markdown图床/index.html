PicGo + Gitee实现markdown图床

自从用上Markdown后,就再也不想用其他的文档了。可是每次自己在将写好的文章上传的网上，遇到了一个问题,图片老是上传不上去,就想着做一个自己的图床,网上找了找教程,然后自己利用码云搭建了一个图层。

准备环境

软件 版本
git 2.24.1.windows.2
node v10.16.0
gitee
PicGo

安装

node

git

PicGo

首先到PicGo官网下载安装包

下载安装完成后打开软件

选择最下面的插件设置，搜索gitee-uploader进行安装（注意这个必须要有nodejs环境才能装）

建立gitee（码云）图床库

注册码云账号很简单，自己到官网注册就行了

登录成功后，点击右上角加号，选择新建仓库

填入仓库名称和路径，然后选择开源

一定要记住勾选使用Readme文件初始化这个仓库，（我之前配置了好几次都没成功，后来百度了一下，发现这个不会自动帮你创建master仓库），最后点击创建。

PicGo配置

打开PicGo，点击图床设置下的gitee

  • repo：用户名/仓库名称，比如我自己的仓库yulinglsy/picBed，也可以直接复制仓库的url

  • branch：分支，这里写上master

  • token：填入码云的私人令牌

  • path：路径，一般写上img，但是我当时没有设置成img，而是直接放在master里

  • customPath和customURL都不用填。在提交到码云后，会显示提交消息，插件默认提交的是 Upload 图片名 by picGo - 时间

创建私人令牌

登录码云，点击自己的头像，再点击设置

点击私人令牌

点击生成新令牌，私人令牌描述随便填，其他的都不用改，点击提交

需要验证登录密码，验证成功后，就会弹出私人令牌

注意：这个令牌只会明文显示一次，建议在配置插件的时候再来生成令牌，直接复制进去，搞丢了又要重新生成一个。

复制粘贴到PicGo粘贴好后，就可以愉快的使用了。

测试

可以选择剪切板上传图片或者直接拖到PicGo上传

直接拖拽上传
